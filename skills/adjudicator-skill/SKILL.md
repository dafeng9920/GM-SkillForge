# Adjudicator Skill: 市场行情感知与决断器
# --------------------------------------------------------
# 🎯 目标：基于本地同步的 snapshot.json，为云端操作提供大环境感知。
# --------------------------------------------------------

## 核心功能 (Market Awareness)

本 Skill 负责监控 `trading_data/snapshot.json` 目录，并根据最新行情调整 Agent 的决策倾向。

### 1. 指标解读逻辑
当读取到行情数据时，自动执行以下分类：

- **行情概览**：读取上证指数、深证成指的涨跌幅。
- **板块热度**：分析 `top_gainers` 列表，识别当前市场的核心主线。
### 3. 决断逻辑 (Adjudication Logic: 意志识别算法)
打分不再仅仅看涨跌，而是结合 **Architecture A+** 同步的深层指标：

- **封单倍率 (Limit-up Strength)**: `封单资金 / 总成交额`。
  - > 10% -> 极强 (得分 +2)
  - < 2% -> 烂板预警 (得分 -1)
- **筹码结构 (Turnover Balance)**:
  - 换手率 [3% - 12%] -> 换手充分，承接力强 (健康加分 +1)
  - 换手率 < 2% -> **流动性陷阱 (一字板/严重缩量)**。
    - *判定*：意志极强但 **不可达成 (Unpurchasable)**。建议：禁止虚拟买入。
  - 换手率 > 20% -> 巨量分歧 (爆头板风险，得分 -2)

- **交易达成度 (Attainability Index)**:
  - `封单资金 / 日成交额` > 100% -> **通道严重拥堵 (一字/T字板)**。
    - *决断*：买入建议降级为“放弃”，不可达成度极高。
  - `连板数` >= 2 且 `换手率` < 3% -> **筹码真空期**。
    - *决断*：极难买入。

### 4. 决断指令
- `!market status`: 报告当前云端已知的最新行情快照。
- `!market adjudicate [代码]`: 执行深度意志识别，给出 [1-10] 的决断分。

## 数据来源 (Data Feed)
本 Skill 本身不具备联网能力，完全依赖 **Architecture A (Local Sourcing)**：本地运行 `fetch_ashare_data.py` 后经由 `cloud_bridge_sync.ps1` 将数据推送到本容器的以下路径：
- **行情文件**：`openclaw-box/workspace/trading_data/snapshot.json`

当用户询问行情时，请务必先使用工具读取该 JSON 文件，然后根据该文件内容进行回答。
