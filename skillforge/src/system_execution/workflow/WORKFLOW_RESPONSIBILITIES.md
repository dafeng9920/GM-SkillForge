# Workflow 职责边界定义

> **证据级别**: PREPARATION | **任务**: T1 | **日期**: 2026-03-19 (返工)

## 核心职责

Workflow 层 **只负责**：

1. **入口编排**: 接收外部请求并路由到正确的处理路径
2. **流程连接**: 连接 orchestrator → service → handler → API 各层
3. **生命周期管理**: 管理请求/响应的完整生命周期，但不参与裁决

## 明确排除的职责

Workflow 层 **不负责**：

| 职责类型 | 说明 | 负责层级 |
|---------|------|---------|
| 治理裁决 | 不做任何 allow/block/deny 决策 | Gate |
| 业务逻辑执行 | 不实现具体业务操作 | Service |
| 资源操作 | 不直接操作数据库/外部系统 | Handler |
| 协议适配 | 不处理 HTTP/WebSocket 等协议 | API |

## 与 Orchestrator 的调用边界

### Workflow → Orchestrator
- **调用方式**: 方法调用，非直接实例化
- **传递内容**: 上下文对象，不传递裁决权
- **返回内容**: 编排结果，不包含治理决策

### 禁止模式
```python
# ❌ 禁止: Workflow 做治理决策
if gate_decision == "ALLOW":
    proceed()

# ✅ 正确: Workflow 只做编排
orchestrator_result = orchestrator.coordinate(context)
# 治理决策由 Gate 层负责
```

## 实现约束

1. **无 Runtime 逻辑**: 只定义接口和骨架，不实现执行逻辑
2. **无外部集成**: 不连接数据库、消息队列、外部 API
3. **无治理语义**: 不引入 gate_decision, permit, adjudication 等概念
4. **只连接不裁决**: 负责连接各层，但不参与任何决策
