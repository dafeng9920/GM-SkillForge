# 云端 Codex 与小龙虾任务分工规则

- 日期：`2026-03-09`

## 一、总原则

凡是“要验证系统链路是否成立”的，给 **云端 Codex**。  
凡是“系统链路已成立后要批量干活”的，给 **小龙虾**。

## 二、云端 Codex 负责什么

适合云端 Codex 的任务：

- 首单流程验证
- 治理底盘安装
- skill / script / bridge / gate 安装与修复
- `SMOKE-GATE` 类联调
- 首个真实受控任务样板
- 边界污染 remediation
- 复杂合规材料整理

特点：

- 纪律性优先
- 产能不是重点
- 边界与状态机必须极细

## 三、小龙虾负责什么

适合小龙虾的任务：

- 单模块
- 低风险
- 文档/报告/规范化类
- 小 patch proposal
- task package 产出
- runtime telemetry 产出

不适合小龙虾的任务：

- 首单流程验证
- 架构设计
- 治理修复
- 高风险代码修改
- 真实 absorb / final accept 相关任务

## 四、当前执行制度

- `REAL-TASK-001`：云端 Codex
- `REAL-TASK-002`：小龙虾
- `REAL-TASK-003`：小龙虾（失败样本）
- `REAL-TASK-003-R1`：云端 Codex

## 五、一句话结论

云端 Codex 是“验证员/底盘执行员”，  
小龙虾是“低风险吞吐执行工人”。
