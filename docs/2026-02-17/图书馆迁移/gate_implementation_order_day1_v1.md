## 1. intake_repo

- **输入前置**: 无
- **通过标准（最小）**: 验证 `repo_url` 格式 + `commit_sha` 格式正确
- **失败即阻断（Fail-Closed）**:
  - `repo_url` 不匹配 `^(https?|git)://.+$` → DENY (`INVALID_REPO_URL`)
  - `commit_sha` 不匹配 `^[a-f0-9]{7,40}$` → DENY (`INVALID_COMMIT_SHA`)

---

## 2. repo_scan_fit_score

- **输入前置**: intake_repo 输出（repo_url + commit_sha 已验证）
- **通过标准（最小）**: 扫描仓库，计算适配度分数 >= 60
- **失败即阻断（Fail-Closed）**:
  - 无法访问仓库 → DENY (`REPO_ACCESS_FAILED`)
  - 适配度分数 < 60 → DENY (`FIT_SCORE_TOO_LOW`)

---

## 3. draft_skill_spec

- **输入前置**: repo_scan_fit_score 输出（扫描结果）
- **通过标准（最小）**: 生成 SkillSpec JSON，包含 skill_id + intent + inputs/outputs
- **失败即阻断（Fail-Closed）**:
  - SkillSpec 缺少必需字段 → DENY (`SKILL_SPEC_INCOMPLETE`)
  - 无法解析仓库结构 → DENY (`REPO_PARSE_FAILED`)

---

## 4. constitution_risk_gate

- **输入前置**: draft_skill_spec 输出（SkillSpec）
- **通过标准（最小）**: 检查 risk_level + side_effects，低风险（tier 0/1）直接通过
- **失败即阻断（Fail-Closed）**:
  - tier >= 2 且无 HITL 审批 → DENY (`HIGH_RISK_NO_HITL`)
  - side_effects 包含未授权操作 → DENY (`UNAUTHORIZED_SIDE_EFFECTS`)

---

## 5. scaffold_skill_impl

- **输入前置**: constitution_risk_gate 输出（风险已评估）
- **通过标准（最小）**: 生成技能脚手架代码，可被 sandbox 加载
- **失败即阻断（Fail-Closed）**:
  - 脚手架生成失败 → DENY (`SCAFFOLD_GENERATION_FAILED`)
  - 代码无法加载 → DENY (`SCAFFOLD_LOAD_FAILED`)

---

## 6. sandbox_test_and_trace

- **输入前置**: scaffold_skill_impl 输出（脚手架代码）
- **通过标准（最小）**: 在沙箱中执行测试，通过率 >= 80%
- **失败即阻断（Fail-Closed）**:
  - 沙箱启动失败 → DENY (`SANDBOX_START_FAILED`)
  - 测试通过率 < 80% → DENY (`TEST_PASS_RATE_TOO_LOW`)
  - 执行超时 → DENY (`SANDBOX_TIMEOUT`)

---

## 7. pack_audit_and_publish

- **输入前置**: sandbox_test_and_trace 输出（测试结果）
- **通过标准（最小）**: 收集所有 Evidence，计算 schema_hash，生成 L3 AuditPack
- **失败即阻断（Fail-Closed）**:
  - Evidence 缺失 → DENY (`MISSING_EVIDENCE`)
  - 无法计算 SHA256 → DENY (`SCHEMA_HASH_COMPUTE_FAILED`)
  - AuditPack 生成失败 → DENY (`AUDIT_PACK_GENERATION_FAILED`)
