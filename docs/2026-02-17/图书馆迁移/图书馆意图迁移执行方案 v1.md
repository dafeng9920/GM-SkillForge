# 图书馆意图迁移执行方案 v1

## 0. 目标
将 `NEW-GM 图书馆` 中的高价值内容意图（包括外螺旋但不限于外螺旋），按可审计标准迁移到 Skill 项目。  
顺序：先确认意图 -> 回落文档 -> Skill 精细化执行。  
硬约束：不迁移代码，只迁移意图与证据。

## 1. 迁移原则（冻结）
1. 只迁“可映射到 Contract/Gate/Evidence/AuditPack/permit”的意图。
2. 只迁“可验证、可回放、可裁决”的能力。
3. 不做整包迁移，不做概念全集复刻。
4. 任一意图未通过准入闸门，直接冻结。

## 2. 迁移范围判定
0. 范围原则：先做“图书馆全量扫描”，再做价值分级与迁移决策。
1. 三大哲学思想：迁（文档层，作为宪法附录/原则）。
2. 53 轴度：分波迁（先高价值 Top 10，参数化接入）。
3. 10 大神器：按不可替代性筛选迁（非刚需冻结）。
4. 四大基石：迁，但必须先完成与现有治理框架映射。
5. 外螺旋相关内容：纳入高价值扫描池，不单独特权，不单独排除。

## 3. AI军团“图书馆审查”任务包（明日执行）
0. 先执行全量目录扫描：覆盖图书馆全部主题域与历史阶段文档。
1. 对每个候选组件输出一条记录：
- `component_id`
- `source_doc_ref`
- `intent_summary`
- `expected_value`
- `mapping_target`（Contract/Gate/Evidence/AuditPack/permit）
- `risk_level`
- `migration_decision`（accept/defer/reject）
- `evidence_ref`
 - `value_score`（high/medium/low）
 - `domain_tag`（outer_spiral / governance / runtime / cognition / tooling / other）
2. 禁止输出“实现方案代码”，只允许意图与映射结论。
3. 每条结论必须附证据引用路径，不接受口头判断。

## 4. 回落文档格式（统一模板）
每个组件回落到单页条目，格式如下：
- 组件名 / 版本快照
- 意图定义（1段）
- 价值判定（高/中/低）
- 映射位置（Contract/Gate/Evidence/AuditPack/permit）
- 失败风险（不迁的后果 / 错迁的后果）
- 迁移决策（accept/defer/reject）
- 证据引用（source_doc_ref + evidence_ref）

## 5. 准入闸门（Intent Admission Gate）
组件仅在以下条件全部满足时进入 Skill 项目：
1. 可形成明确输入/输出契约。
2. 可生成 EvidenceRef。
3. 可纳入 Gate 判定（PASSED/REJECTED）。
4. 可进入 L3 AuditPack。
5. 不引入 n8n 裁决权。

任一条件不满足：`defer/reject`。

## 6. 执行节奏（建议）
1. Day-1：图书馆审查与回落（不改实现）。
2. Day-2：冻结首批迁移清单（Top 10 轴 + 必要基石）。
3. Day-3：在 Skill 项目建立 contracts/rubric/regression 草案。
4. Day-4+：按 Gate 流程精细化实施与验收。

## 7. 交付物清单
1. `library_intent_inventory_v1.md`（组件审查总表）
2. `library_intent_mapping_v1.md`（映射矩阵）
3. `library_migration_batch_top10_v1.md`（首批迁移清单）
4. `skill_execution_backlog_v1.md`（Skill 项目执行队列）

## 8. 失败防线
1. 禁止“先实现后补治理”。
2. 禁止“无证据迁移”。
3. 禁止“未 permit 发布”。
4. 禁止“n8n 代替 SkillForge 裁决”。
