# 前端基线方案评审提示词

> 版本: v1.0
> 创建日期: 2026-02-20
> 用途: 前端基线方案评审

---

## 使用说明

将以下提示词发送给 AI（产品设计师 + 前端架构师角色），用于评审前端基线方案。

---

## 提示词内容

```
你是资深产品设计师 + 前端架构师。请基于以下上下文，输出"可落地的前端基线方案评审意见"。

【项目上下文】
- 项目：SkillForge L4.5
- 已有后端能力：run_intent / fetch_pack / query_rag / import_external_skill / health
- 安全边界：n8n 只能触发与路由；最终裁决由 SkillForge 给出
- 关键返回字段：run_id, evidence_ref, gate_decision, release_allowed
- 文档基线：FRONTEND_REQUIREMENTS_v1.md（v1.3）

【候选基线方案（待评审）】
- 页面：8页
  1) /n8n/run-intent
  2) /skills/import
  3) /governance/permits
  4) /n8n/audit-packs
  5) /n8n/rag-query
  6) /governance/gates
  7) /governance/work-items
  8) /n8n/health
- 布局：左侧导航 + 顶栏 + 主内容；三类模板（表单执行、列表详情、监控仪表）
- 风格：中文优先、浅色工业风、Noto Sans SC + JetBrains Mono、8px spacing、圆角10、状态色固定（ALLOW绿/BLOCK红/WARN橙）

【你的任务】
请输出：
1) 你是否同意该基线（同意/部分同意/不同意）
2) 必改项（最多5条，必须具体到页面或组件）
3) 风格优化建议（最多5条，避免空泛）
4) 信息架构风险（最多3条）
5) 最终建议页面数量与分组（给出你建议的版本）
6) 用 0-100 分打分：可用性、可实施性、一致性、可扩展性
7) 给出你建议的"冻结版本 v1.0"摘要（不超过200字）

【输出格式要求】
- 必须用中文
- 用 Markdown
- 先给结论，再给明细
- 不要复述上下文
- 不要输出代码

------------

【视觉风格（Style）】
方案A：工业控制台（浅灰+蓝绿，专业稳）
方案B：审计中台（白底高对比，证据导向）
方案C：科技仪表盘（深浅分层，状态突出）

【产品格调（Tone）】
严谨可审计：少情绪文案，多证据字段
操作透明：每一步都给 run_id/evidence_ref
Fail-Closed 友好：报错明确"为什么被拦截、怎么修"

【UI 设计规则（Design Rules）】
信息层级：先结论（ALLOW/BLOCK）再细节
状态统一：ALLOW=绿，BLOCK=红，PENDING=灰，WARN=橙
卡片结构统一：标题区 / 输入区 / 结果区 / 证据区
表单统一：左标签右输入，错误贴字段，不弹全局模糊报错

【展示方式（Presentation）】
Run Intent：上表单，下执行结果时间线
Import Skill：6步流水线横向进度 + 每步详情抽屉
Gate Dashboard：流程图 + Gate 明细表（耗时/错误码/证据）
AuditPack：左列表右详情（JSON + replay_pointer）
RAG：查询区 + 命中片段 + 回放指针
```

---

```markdown

1. [具体页面/组件]：[问题描述] → [修改建议]
2. ...

### 3. 风格优化建议（最多5条）
1. [具体建议]
2. ...

### 4. 信息架构风险（最多3条）
1. [风险描述] → [缓解措施]
2. ...

### 5. 最终建议页面数量与分组
- 页面数量：__ 页
- 分组建议：
  - 组1：xxx（包含页面 a, b, c）
  - 组2：xxx（包含页面 d, e）
  - ...

### 6. 打分（0-100）
| 维度 | 分数 | 说明 |
|------|------|------|
| 可用性 | __ | ... |
| 可实施性 | __ | ... |
| 一致性 | __ | ... |
| 可扩展性 | __ | ... |

### 7. 冻结版本 v1.0 摘要（不超过200字）
[摘要内容]
```
