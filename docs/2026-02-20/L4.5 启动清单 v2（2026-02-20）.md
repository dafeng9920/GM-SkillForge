# L4.5 启动清单 v2（2026-02-20）

> 阶段定位：L4 已收口，进入 L4.5（n8n 编排接入 + 外部 Skill 治理导入）
> 阶段目标：在不破坏宪法边界的前提下，完成“可编排、可裁决、可追责”的最小闭环。

---

## 0. 执行总则（不可回退）

1. n8n 只做编排（触发/路由/重试/通知），不做最终裁决。
2. SkillForge 负责最终裁决（GateDecision / Permit / EvidenceRef / AuditPack）。
3. no-permit-no-release 永不放松。
4. tombstone 默认不可见，仅 `at-time` 可回放审计。
5. 外部 Skill 禁止直接入主仓，必须先走隔离导入流程。

## 1. 阶段目标（Day-1）

### 1.1 主线A：n8n 编排接入（最小闭环）
- 打通三接口：`run_intent`、`fetch_pack`、`query_rag(at_time)`。
- 跑通 1 条真实工作流（固定输入：`repo_url + commit_sha + at_time`）。
- 输出必须包含：`run_id`、`gate_decision`、`evidence_ref`、`replay_pointer`（如有）。

### 1.2 主线B：外部 Skill 治理导入（最小闭环）
- 固化并执行状态机：`Import -> Quarantine -> Constitution -> L1~L5 Audit -> Decision -> Permit -> Registry Admission`。
- 必须支持两条硬规则：
  - Upgrade：同流程，禁止覆盖，只增 revision。
  - Harmful：`Tombstone`，默认查询不可见，审计可回放。

### 1.3 主线C：对话即交付（Plan→Contract→Proof）
- **开放窗口**：需求澄清与 intent 合同化
- **工作窗口**：执行与回执
- **输出**：可直接在 VSCode 落地的"蓝图包"
- 跑通完整链路：`需求输入 → 三技能增强 → 蓝图包生成 → VSCode 落地`

## 2. 边界冻结（本阶段硬门）

### 2.1 角色边界
- n8n 允许：trigger / route / retry / notify。
- n8n 禁止：final decision / release override / direct registry admission。
- SkillForge 必须：final gate decision + permit validation + evidence writeback。

### 2.2 输入边界
- 每次执行必须固定：`repo_url`、`commit_sha`、`at_time`。
- 禁止 “latest 自动漂移输入”。

### 2.3 证据边界
- 每次执行都必须落证据：`run_id`、`evidence_ref`、`gate_decision`、`audit_pack_ref`（通过时）。

## 8. 核心亮点：对话即交付（Plan+Contract+Proof）

### 8.1 价值定位
> **L4.5 不只打通编排，还要把”对话即交付（Plan+Contract+Proof）”变成可验收能力。**

用户从开放对话窗口输入需求 → 三技能加持增强 → 工作窗口执行 → 输出全套蓝图包。

**差异点**：
- 市面产品：聊完得到文本建议，执行靠用户自己拼
- 本系统：聊完直接产出”可执行蓝图包”，可追踪、可复盘、可交付

### 8.2 三段式产物标准（必交付）

| 产物 | 内容 | 消费方 |
|------|------|--------|
| **Plan** | 10阶结构化蓝图（理解→拆解→计划→依赖→风险→验收） | 用户可见 |
| **Contract** | 执行合约（可机器读取，给工作窗口/n8n消费） | 执行臂 |
| **Proof** | 证据摘要（run_id / evidence_ref / permit / replay） | 审计方 |

### 8.3 格式稳定性硬门
- 输出必须命中固定 schema，禁止散文漂移
- 未命中直接 fail-closed，不进入执行窗口

### 8.4 验收指标
| 指标 | 定义 | 目标 |
|------|------|------|
| 蓝图包完整率 | Plan/Contract/Proof 三件套齐全率 | ≥95% |
| 可执行通过率 | Contract 可被执行臂消费 | ≥90% |
| 可复核通过率 | Proof 可回放 | 100% |

### 8.5 交付包标准（必交付）

蓝图包必须包含以下文件：

```
project_blueprint/
├── SPEC.md                    # 规格说明（需求边界、功能点）
├── ARCHITECTURE.md            # 架构设计（模块划分、依赖关系）
├── TASKS.md                   # 任务拆解（可执行清单）
├── intent_*.json              # 意图合同（结构化 intent 定义）
├── tests/
│   └── acceptance/*           # 验收测试用例
├── README.md                  # 项目说明（快速上手）
└── AuditPack/                 # 审计包
    ├── evidence/              # 证据文件
    ├── provenance/            # 来源追溯
    ├── run_id                 # 执行标识
    ├── evidence_ref           # 证据引用
    └── replay_pointer         # 回放指针
```

### 8.6 验收用例（L4.5 必跑）

| 用例 | 描述 | 预期结果 |
|------|------|----------|
| 真实场景 | "贷款利率计算软件" | 从开放窗口到工作窗口全链跑通 |
| 输出验证 | 蓝图包完整性检查 | 产出完整工程包 + run_id/evidence_ref/replay_pointer |
| 格式验证 | Schema 命中检查 | 所有文件符合合同 schema |
| 回放验证 | at-time 复现 | 同输入可复现相同输出 |

### 8.7 质量门（Fail-Closed）

| 门禁 | 规则 | 失败处理 |
|------|------|----------|
| Schema 命中 | 输出必须命中合同 schema | 直接 fail-closed，不进入执行窗口 |
| 格式漂移 | 禁止散文式输出 | 阻断 + 告警 |
| FAIL 产出 | 即使失败也要产出 required_changes | 返回整改清单，不静默失败 |

## 12. 社区市场：SkillPack 交易规则（L4.5 硬门）

### 12.1 核心定位

> **我们卖的不是 skill，是 L3 级可复核、可放行、可升级的 SkillPack。**

**分水岭**：起步即 L3，与市面上"能跑就行"的 skill 天然区隔。

### 12.2 上架硬门（4条不可回退）

| 规则 | 说明 | 违反处理 |
|------|------|----------|
| **L3 才能上架** | 必须 PASS@L3 + permit + AuditPack | 拒绝上架 |
| **交易对象=SkillPack** | 卖的是 revision + pack_hash，不是 repo | 版本不认 |
| **升级必须重审** | 更新走完整治理流程，只增 revision | 禁止覆盖 |
| **有害即 tombstone** | 下架但保留历史，at-time 可回放 | 不可删除 |

### 12.3 会员分层（创作者侧）

| 层级 | 权益 | 价格 |
|------|------|------|
| **Free** | 少量审计额度、1 个上架位、基础商品页 | 免费 |
| **Pro** | 更多 L3 审计额度、优先队列、自动升级重审、模板库、数据看板 | 订阅制 |
| **Studio/Team** | 团队协作、组织资产、RBAC、私有上架、企业交付支持 | 高级订阅 |

### 12.4 分成规则

| 模式 | 创作者 | 平台 | 说明 |
|------|--------|------|------|
| 自托管包（下载/导入） | 85% | 15% | 平台成本低 |
| 托管执行（代跑） | 70% | 30% | 平台承担算力/运维/风险 |
| 企业版（白名单/审计） | 另议 | 订阅制 | 主要收入来源 |

### 12.5 反腐败规则（必须定死）

| 规则 | 说明 |
|------|------|
| **会员 ≠ 过审** | 会员买的是工具/额度/优先级，不是 PASS |
| **审计独立** | L3 审计结果不受会员等级影响 |
| **permit 与 pack_hash 强绑定** | 卖的是"这一版"，不是"这个 repo" |
| **license/归属声明** | 上架必须声明许可与来源，否则拒绝 |

### 12.6 商品页标准（自动生成）

每个 SkillPack 商品页必须包含：
- 版本：`repo@commit → revision`
- 证据摘要：关键 findings / EvidenceRef（可点开）
- AuditPack 摘要：L1~L5 通过状态
- 变更记录：evolution.json 摘要
- 状态：PASS / tombstoned

### 12.7 最小 MVP 路线

```
Phase 1（L4.5）:
├── 创作者：提交 repo@commit → 跑 L3 → PASS 上架 / FAIL 给 required_changes
├── 买家：购买 → 下载 SkillPack + permit 证明
└── 平台：tombstone/升级重审可用

Phase 2（v1）:
├── 托管执行（可选）
└── 企业白名单与组织治理

Phase 3（v2）:
└── 买家订阅、批量导入、企业留档
```

### 12.8 冷启动策略

1. **标杆 SkillPack**：先做 3-5 个 L3 级示范作品
2. **创作者模板**：contracts 模板 + controls 默认项 + tests 示例 + 一键 CLI
3. **免费层**：允许试跑 1 次 L3 或上架 1 个商品，形成供给增长

## 13. 内螺旋：质量增强层（L4.5 可选）

### 13.1 角色定义

| 螺旋 | 方向 | 职责 |
|------|------|------|
| **外螺旋** | 对外 | 编排与交付（n8n/SkillPack/社区） |
| **内螺旋** | 对内 | 校正与提纯（反思、纠偏、降幻觉、提证据） |

### 13.2 接入位置

```
Generate ──► [内螺旋介入] ──► Adopt ──► [内螺旋复核] ──► Execute
                │                          │
                ▼                          ▼
           内审一次                    风险复核一次
```

### 13.3 内螺旋产物（最小输出）

| 产物 | 说明 |
|------|------|
| `self_check_report` | 自检报告 |
| `risk_deltas` | 风险增量识别 |
| `evidence_gap_list` | 证据缺口清单 |
| `recommended_changes` | 建议整改项 |

### 13.4 硬规则

| 规则 | 说明 |
|------|------|
| **非裁决性** | 内螺旋只能"建议/阻断建议"，不能绕过 Gate 直接放行 |
| **最终裁决归 Gate** | 所有 release_decision 必须由 SkillForge Gate 给出 |
| **可选但可追溯** | 内螺旋调用记录必须进入 EvidenceRef |

- 跑 1 条"内螺旋介入前后对比"链路
- 验证输出质量提升（evidence_gap 减少）
- 验证不破坏 E001/E003 语义（阻断仍由 Gate 控制）
