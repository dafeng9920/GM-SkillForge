## 一、执行目标

只做两件事：

1. 收干净 Git 仓库里的明显噪音和运行时杂质
2. 把不属于 SkillForge 主线的业务从主线判断里剥离出来

本单不是：

- 全仓历史证据补录工程
- T4/T5 全量补 gate 工程
- 全仓一次性大迁仓工程

## 二、执行原则

- 先分类，后执行
- 先低风险清理，后处理中风险剥离
- 不因为治理过载而阻塞仓库清理
- 不把“剥离候选”写成“已经迁移完成”

## 三、P0：立即执行

### P0-1 Git 忽略与运行时杂质收口

目标：

- 把明显不该进仓库的运行时/缓存/敏感文件收口进 `.gitignore`

范围：

- `.env`
- `.env.bak`
- `db/*.sqlite`
- `trading_data/`
- `demo_*_data/`
- `.tmp/`
- `.pytest_cache/`
- `dist/`
- `logs/`
- `reports/ALL_TESTS_PASSED.flag`
- `GM-SkillForge.tmppytest/`

预期结果：

- 后续不再持续污染 `git status`

### P0-2 根目录散落文件分类

目标：

- 把根目录散落的 HTML、报告、补丁脚本、测试脚本、临时脚本分流

分流规则：

- 开发脚本：移入 `scripts/dev/`
- 审计/报告产物：移入 `reports/` 或 `audit/`
- SEO 页面：并入 `pseo/export-seo` 待剥离范围
- 明显临时测试文件：进入删除候选清单

重点对象：

- `Audit_Report_*.html`

- `nextjs-ai-seo-checker.html`
- `shopify-ai-seo-checker.html`

- `create_valid_dummy_packs.py`
- `insert_debug.py`

- `check_institution.py`

### P0-3 非 skill 主线业务冻结为剥离候选

## 六、执行顺序

1. 先改 `.gitignore`
2. 再整理根目录散落文件
3. 再冻结非 skill 业务边界
4. 最后再决定是否进入独立仓迁移

## 七、完成标准

满足以下条件即可算本轮完成：

- `.gitignore` 已覆盖主要运行时污染项
- 根目录散落文件完成第一次分类
- `quant / pseo` 两条线被正式标记为非主线业务
- 后续主线判断不再被这些业务混入

一句话：

- 先把仓库收干净，把边界划清，再决定迁仓，不做治理过载。
