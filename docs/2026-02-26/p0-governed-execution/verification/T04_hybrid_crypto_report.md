## 3. 实现概要

### 3.1 混合加密流程

```
加密流程:
1. 生成随机 DEK (32 bytes for AES-256)
2. 使用 AES-256-GCM 加密 body → (iv, ciphertext, tag)
3. 使用 RSA-OAEP-256 加密 DEK → encrypted_dek
4. 返回 EncryptionResult(encrypted_dek, iv, ciphertext, tag)

解密流程:
1. 使用 RSA-OAEP-256 解密 encrypted_dek → DEK
2. 使用 AES-256-GCM 解密 (iv, ciphertext, tag) + DEK → plaintext
3. 验证 tag (任何篡改都会导致失败)
```

### 3.2 核心函数

| 函数 | 描述 |
|------|------|
| `generate_dek()` | 生成 32 字节随机 DEK |
| `generate_rsa_keypair()` | 生成 RSA 密钥对 (默认 2048 bits) |
| `encrypt_dek(dek, public_key)` | RSA-OAEP-256 加密 DEK |
| `decrypt_dek(encrypted_dek, private_key)` | RSA-OAEP-256 解密 DEK |
| `encrypt_body(plaintext, dek)` | AES-256-GCM 加密 body |
| `decrypt_body(iv, ciphertext, tag, dek)` | AES-256-GCM 解密 body |
| `hybrid_encrypt(plaintext, public_key)` | 高层加密 API |
| `hybrid_decrypt(result, private_key)` | 高层解密 API |

### 3.3 数据结构

```python
@dataclass
class EncryptionResult:
    encrypted_dek: str  # Base64 编码的 RSA 加密 DEK
    iv: str             # Base64 编码的 IV (12 bytes)
    ciphertext: str     # Base64 编码的密文
    tag: str            # Base64 编码的认证标签 (16 bytes)
    key_id: str         # 可选的密钥标识符
```

## 6. 与 T03 (Envelope) 集成

EncryptionResult 可直接用于 EvidenceEnvelopeBuilder:

```python
enc_result = hybrid_encrypt(body, public_key, key_id="rsa-001")

envelope = (
    EvidenceEnvelopeBuilder()
    .with_node_id("node-001")
    .with_encryption(
        encrypted_dek=enc_result.encrypted_dek,
        key_id=enc_result.key_id,
        ciphertext_data=enc_result.ciphertext,
        ciphertext_iv=enc_result.iv,
        ciphertext_tag=enc_result.tag,
    )
    .build()
)
```
