# L4.1 用户开放前 20 条必须项（2026-02-26）

目标：从“自己可用”升级到“可对用户开放的最小安全基线”。  
规则：`P0` 任一未达成 => `DENY`；`P1` 任一未达成 => `REQUIRES_CHANGES`。

## L4.1 节奏

1. Phase A（P0）：先封住高风险入口（必须全过）
2. Phase B（P1）：再补用户运营与治理闭环（必须全过）
3. Phase C：通过后才允许灰度开放

---

## P0（阻断级，10 条）

1. 统一身份认证（登录 + 会话过期）
- 验收：未登录不可调用核心 API。

2. RBAC 最小权限模型（Admin/Operator/User）
- 验收：低权限不能触发高风险动作。

3. 租户隔离（tenant_id 强制注入与校验）
- 验收：跨租户读写被拒绝并审计。

4. API 速率限制与配额
- 验收：超限返回标准错误码，不可绕过。

5. 高风险动作二次确认（删除/发布/覆盖）
- 验收：无确认 token 不执行。

6. Fail-Closed 执行门
- 验收：治理系统异常时默认拒绝，不走 fail-open。

7. 输入安全防护（注入/越权字段拦截）
- 验收：命中策略返回阻断码并记录 evidence_ref。

8. 审计日志不可篡改链（最小 hash 链）
- 验收：每次关键操作可追溯 `actor/action/time/evidence_ref`。

9. 敏感配置与密钥管理
- 验收：无明文密钥入库；关键密钥可轮换。

10. 紧急熔断与回滚开关
- 验收：单命令可停写/停发并回滚到最近稳定版本。

---

## P1（上线前，10 条）

11. 用户可见错误分级与提示规范
- 验收：错误信息区分用户提示与开发诊断。

12. 申诉与人工复核入口
- 验收：被拒绝任务可提交复核并留痕。

13. 数据保留与删除策略（含保留期限）
- 验收：支持按策略清理并可审计。

14. 隐私与合规声明落地（最小版）
- 验收：前端可见并与实际行为一致。

15. 操作告警（异常频率/失败率/越权尝试）
- 验收：阈值触发告警并进入事件流。

16. 灰度发布与回滚演练
- 验收：至少 1 次灰度 + 1 次回滚演练通过。

17. 用户级限流与风控策略
- 验收：高风险用户触发限制，正常用户不误伤。

18. 前端最小防护（CSRF/XSS 基线）
- 验收：关键页面通过基线扫描。

19. 证据包导出与第三方可验
- 验收：用户可导出最小证据并可独立校验。

20. 上线 Gate 自动化
- 验收：合并/发布前自动检查 1-19，失败即阻断。

---

## 裁决规则（L4.1 Final Gate）

1. 缺任一 P0：`DENY`
2. P0 全过但缺任一 P1：`REQUIRES_CHANGES`
3. P0/P1 全过：`ALLOW`（允许灰度开放）
