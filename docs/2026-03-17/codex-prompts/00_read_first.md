## 你当前必须理解的核心事实

### 1. 系统已经拆成两条线
当前系统不是单线架构，而是：

- **生成线 / 生产主线**
- **审计线 / 治理内核**

两者不能混为一体。

---

### 2. 当前阶段策略已经确定
当前阶段不是双线同时扩建，而是：

- **治理线冻结为基线，只修不扩**
- **生产线转为主攻方向**
- **后续通过 candidate / validate / release / audit 再重新合拢**

---

### 3. 当前唯一受保护的生产主链路
你必须围绕以下主链路理解所有创建文档：

`需求输入 → Intent Draft → Contract Draft → Candidate Skill → Build Validation → Delivery Manifest → skill-creator / 打包层`

任何整合都不得破坏这条主链。

---

### 4. skill-creator 的角色已经被限定
`skill-creator` 不是完整创建流程，不是主创建引擎，不是审计器，也不是裁决器。

它只负责后段：

- 结构整理
- 标准 skill 目录封装
- `SKILL.md` / 资源归档
- 最终 `skill.zip` 打包交付

---

### 5. 审计与打包的顺序已经确定
当前口径是：

- **主审计在打包前**
- **打包后只允许做轻量交付审计**

不得反转顺序。

---

### 6. 最终裁决权不属于 Skill，不属于 n8n，不属于 skill-creator
以下对象都不得拥有最终裁决权：

- Skill
- workflow
- n8n
- orchestrator
- scheduler
- skill-creator

最终裁决权只属于：

- Kernel / Governor / 治理内核
