## 2. 开发原则（实现阶段）

硬原则：

* 先落主链，再补支撑页
* 先落壳层与静态结构，再接数据契约
* 先确保 Never-in-DOM / API 裁剪守住，再谈联调
* 页面实现必须服从已过 Final Gate 的规格，不得即兴改叙事

当前优先级：

1. `Audit Detail`
2. `Permit`
3. `Dashboard`
4. 共享治理组件
5. 权限裁剪守卫
6. Landing Page / CTA 收口
7. `Registry / History / Gap Analysis`

---

## 3. 三权分立（MUST）

每个任务必须存在三种独立权力，不得混同：

* `Execution`：按合同写代码，无放行权
* `Review`：检查实现质量、偏差与回归风险
* `Compliance`：按 B Guard 做硬拦截

执行顺序固定为：

1. Review 完成审查意见
2. Compliance 产出 `ComplianceAttestation`
3. 仅当 `Compliance=PASS` 时，Execution 才能进入落地代码阶段

## 6. 统一 DoD（实现阶段）

全部任务完成后，必须同时满足：

1. 旧 `execute / audit / system` 导航结构已被治理主链 IA 替换
2. `Dashboard / Audit Detail / Permit` 具备真实页面壳层与主要模块实现
3. 共享组件已从页面内重复结构中抽出

5. 首页文案与应用内 CTA 已切换为治理叙事
