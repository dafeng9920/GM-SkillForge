## 2. 方案对比

### 2.1 DB 方案（传统数据库）

| 维度 | 评估 |
|------|------|
| 实现复杂度 | ✅ 低 |
| 查询灵活性 | ✅ 高 |
| 不可变性 | ❌ 需额外约束 |
| 审计追溯 | ⚠️ 依赖应用层 |
| 水平扩展 | ⚠️ 需分片 |
| 合规性 | ⚠️ 需额外证明 |

### 2.2 Append-Only Log 方案

| 维度 | 评估 |
|------|------|
| 实现复杂度 | ⚠️ 中 |
| 查询灵活性 | ⚠️ 中 |
| 不可变性 | ✅ 天然支持 |
| 审计追溯 | ✅ 天然支持 |
| 水平扩展 | ✅ 天然支持 |
| 合规性 | ✅ 满足要求 |

### 2.3 决策矩阵

| 权重 | 维度 | DB | Append-Only Log |
|------|------|----|-----------------|
| 30% | 不可变性 | 3 | 10 |
| 25% | 审计追溯 | 5 | 10 |
| 20% | 合规性 | 5 | 10 |
| 15% | 实现复杂度 | 10 | 5 |
| 10% | 查询灵活性 | 10 | 5 |

**加权得分**:
- DB: 3×0.3 + 5×0.25 + 5×0.2 + 10×0.15 + 10×0.1 = **5.25**
- Append-Only Log: 10×0.3 + 10×0.25 + 10×0.2 + 5×0.15 + 5×0.1 = **8.25**

**结论**: 选择 **Append-Only Immutable Log**

## 4. 保留期（Retention）

### 4.1 保留策略

| 数据类型 | 保留期 | 压缩策略 |
|----------|--------|----------|
| Tombstone 记录 | **7 年** | 不压缩 |
| Tombstone 索引 | 7 年 | 可压缩 |
| Tombstone 元数据 | 7 年 | 可压缩 |

### 4.2 保留期依据

| 依据 | 要求 |
|------|------|
| 合规要求 | 金融/安全审计 ≥ 7 年 |
| 回滚能力 | 保留期内可追溯 |
| 法律要求 | 争议处理需要历史记录 |

### 4.3 过期处理

```
超过保留期 → 标记为 EXPIRED → 移动到冷存储 → 最终删除
```

- **标记延迟**: 24 小时
- **冷存储迁移**: 7 天
- **最终删除**: 30 天后

## 5. 回放一致性要求

### 5.1 一致性保证

| 保证级别 | 描述 |
|----------|------|
| **写后读一致性** | 写入成功后立即可读 |
| **单调读** | 读取不会看到更旧的数据 |
| **因果一致性** | 相关操作按因果顺序可见 |

### 5.2 回放验证

| 验证项 | 要求 | 验证方法 |
|--------|------|----------|
| replay_pointer 有效性 | 100% | 签名校验 |
| 数据完整性 | 100% | Hash 校验 |
| 顺序正确性 | 100% | 序号校验 |
| 无数据丢失 | 100% | 完整性校验 |

### 5.3 回放时间要求

| 场景 | 时间要求 |
|------|----------|
| 单条记录回放 | < 10ms |
| 批量回放（1000 条） | < 1s |
| 全量回放 | < 10 分钟 |

## 6. 不可变性策略

### 6.1 技术实现

| 层级 | 策略 | 实现 |
|------|------|------|
| 存储层 | Append-Only | Kafka / Kinesis / EventStore |
| 访问层 | 只读 API | 禁止 UPDATE/DELETE |
| 校验层 | 签名校验 | 每条记录带签名 |
| 审计层 | 访问日志 | 所有读写操作记录 |

### 6.2 防篡改机制

| 机制 | 描述 |
|------|------|
| **Hash 链** | 每条记录包含前一条的 Hash |
| **签名** | 每条记录带数字签名 |
| **时间戳** | 不可篡改的时间戳服务 |
| **多副本** | 跨区域多副本存储 |

### 6.3 链式结构

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Record N   │───>│  Record N+1 │───>│  Record N+2 │
├─────────────┤    ├─────────────┤    ├─────────────┤
│ prev_hash   │    │ prev_hash   │    │ prev_hash   │
│ data        │    │ data        │    │ data        │
│ signature   │    │ signature   │    │ signature   │
│ timestamp   │    │ timestamp   │    │ timestamp   │
└─────────────┘    └─────────────┘    └─────────────┘
```

## 7. 数据模型

### 7.1 Tombstone 记录结构

```json
{
  "tombstone_id": "TOMB-20260218-001",
  "run_id": "RUN-20260218-CANARY-001",
  "permit_id": "PERMIT-20260218-001",
  "release_id": "REL-20260218-CANARY-001",
  "target": {
    "repo_url": "github.com/example/repo",
    "commit_sha": "a1b2c3d4..."
  },
  "status": "ROLLED_BACK",
  "created_at": "2026-02-18T11:00:00Z",
  "created_by": "rollback-system",
  "prev_hash": "sha256:abc123...",
  "signature": "SHA256:def456...",
  "metadata": {
    "rollback_reason": "simulated-error",
    "rollback_duration_ms": 95000
  }
}
```

### 7.2 索引结构

| 索引名 | 字段 | 类型 |
|--------|------|------|
| idx_run_id | run_id | B-Tree |
| idx_permit_id | permit_id | B-Tree |
| idx_created_at | created_at | B-Tree |
| idx_target | target.repo_url, target.commit_sha | Composite |

## 8. API 规范

### 8.1 写入 API

```
POST /api/v1/tombstones
Content-Type: application/json
Authorization: Bearer <service_token>

Request:
{
  "run_id": "RUN-20260218-CANARY-001",
  "permit_id": "PERMIT-20260218-001",
  "release_id": "REL-20260218-CANARY-001",
  "target": { ... },
  "status": "ROLLED_BACK"
}

Response:
{
  "tombstone_id": "TOMB-20260218-001",
  "created_at": "2026-02-18T11:00:00Z",
  "prev_hash": "sha256:abc123...",
  "signature": "SHA256:def456..."
}
```

### 8.2 查询 API

```
GET /api/v1/tombstones?run_id=RUN-20260218-CANARY-001

Response:
{
  "tombstones": [
    { "tombstone_id": "TOMB-20260218-001", ... }
  ],
  "pagination": { ... }
}
```

### 8.3 验证 API

```
POST /api/v1/tombstones/verify
Content-Type: application/json

Request:
{
  "tombstone_id": "TOMB-20260218-001",
  "verify_chain": true
}

Response:
{
  "valid": true,
  "chain_verified": true,
  "signature_valid": true
}
```
