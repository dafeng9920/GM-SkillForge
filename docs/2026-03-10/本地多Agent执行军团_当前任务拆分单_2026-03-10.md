# 本地多Agent执行军团 当前任务拆分单

- 日期：`2026-03-10`
- 参考规范：`multi-ai-collaboration.md`
- 当前主线：`本地多Agent协同建设`
- mode：`strict`

## 一、目标

把当前主线从“云端执行验证”切回“本地多Agent系统建设”，并按三权分立方式拆成可并行推进的任务。

## 二、当前任务范围

本轮只处理 4 类主线事项：

1. 主链接口与运行链盘点
2. 当前代码/脚本待落地点梳理
3. 验证与门禁缺口梳理
4. 最小治理文档同步

## 三、角色分工

- `Orchestrator`：Codex / 主控
- `Execution`：本地多 Agent 执行臂
- `Review`：独立审查 Agent
- `Compliance`：独立合规 Agent

硬规则：

- `Review` 与 `Execution` 不得同体
- `Compliance` 不参与具体实现
- 无 `Compliance PASS` 不进入真实副作用执行

## 五、任务定义

### T-LOCAL-01 主链盘点

- 目标：盘点当前本地主链 `Permit -> pre_absorb_check -> absorb -> local accept -> final accept` 在仓内的真实落点
- 输出：主链节点、对应脚本/文档、当前缺口
- 限制：只盘点，不改代码

### T-LOCAL-02 实现落点梳理

- 目标：梳理当前最值得优先落地的代码/脚本/接口点
- 输出：按优先级排序的落点清单
- 限制：不直接修改业务逻辑

### T-LOCAL-03 验证缺口梳理

- 目标：梳理当前验证、测试、门禁还缺什么
- 输出：最小验证缺口清单
- 限制：不直接宣称通过

### T-LOCAL-04 汇总执行单

- 目标：把前三项汇总成一份本地主线执行单
- 输出：下一阶段本地多Agent主线执行单
- 限制：必须等待前三项完成

## 六、放行顺序

### Wave 1 放行条件

- `T-LOCAL-01` 完成 Review
- `T-LOCAL-02` 完成 Review
- `T-LOCAL-03` 完成 Review
- 三项全部获得 `Compliance PASS`

### Wave 2 放行条件

- `T-LOCAL-01` = `ALLOW`
- `T-LOCAL-02` = `ALLOW`
- `T-LOCAL-03` = `ALLOW`

## 七、一句话执行口令

先把本地主线盘清、缺口列清、优先级排清，再进入真正的多Agent并行落地；没有 `Compliance PASS` 不推进真实副作用执行。
