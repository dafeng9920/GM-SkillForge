# REAL-TASK-004 任务定义

- 日期：`2026-03-10`
- 任务ID：`REAL-TASK-004`
- 执行者：`云端小龙虾`
- 任务性质：`低风险 / 单模块 / 强约束 / 可审计`
- 任务目标：完成环境一致性治理收口，并验证 `REAL-TASK-002` 执行范本是否可被稳定复用

## 一、任务定位

`REAL-TASK-004` 不是为了再证明 `REAL-TASK-002` 做对了什么，  
而是为了在一个仍然低风险、但有实际治理价值的对象上验证：

- 已经跑通的执行链
- 已经形成的记录结构
- 已经固化的验收方式

是否可以在下一单中稳定复用且不变形。

本单具体承担的治理作用是：

- 收口 `/root/openclaw-box/...` 与 `/home/node/.openclaw/...` 的环境口径分叉
- 明确后续主执行环境
- 把环境差异从“反复讨论点”变成“已固化结论”

## 二、任务边界

本单继续限制为：

- 低风险
- 单模块
- 文档/规范化类
- 不碰高风险代码
- 不碰跨模块改动

本单允许触达的对象固定为：

- 新增：`docs/2026-03-10/环境一致性治理结论_2026-03-10.md`
- 更新：`docs/2026-03-09/NEXT_START_TODO_2026-03-09.md`

## 三、复用目标

本单必须复用 `REAL-TASK-002` 已验证通过的最小执行链：

- `GO`
- `pre_absorb_check`
- `absorb`
- `local_accept`
- `final_accept`

本单也必须复用相同的最小记录结构：

- `completion_record`
- `test_report`
- 执行单
- 验收结论

## 四、成功标准

若本单完成后同时满足以下条件，则视为复用成功：

- 执行链无变形
- 状态词无污染
- 任务范围未漂移
- 证据链完整
- 最终完成 `absorb + final_accept`
- 已明确哪一套是后续主执行环境
- 已明确另一套环境仅作历史/背景记录或治理遥测记录

## 五、失败判定

任一情况出现即视为失败：

- 任务边界扩大
- 状态词回退混乱
- 复用链路断裂
- 证据链缺失
- 未完成真实 `absorb + final_accept`
- 环境口径结论仍然模糊

`REAL-TASK-004` 的作用，是用“环境一致性治理收口”这件有实际作用的低风险任务，
