# AI 会议提示词｜军团任务投递与自动流转协议 v1

## 用途

把这份提示词直接发给 Claude，要求它从外部视角审查当前流程，并给出可执行建议。

本轮目标不是让 Claude 直接改代码，而是让它回答：
- 这套主控官 + AI 军团流程是否合理
- 哪些地方最消耗主控官体力
- 哪些环节最适合协议化 / 自动化
- 应该先做 skill、协议文档，还是代码工具
- 军团任务投递与自动流转协议应该怎么设计

---

## 发给 Claude 的提示词

```text
你现在不是来直接写业务代码。

你要参加一场“AI 会议式评审”，主题是：

- GM-SkillForge 的主控官 + AI 军团协作流程

你的任务是：
- 从专业架构与流程设计角度
- 审查这套流程是否合理
- 找出最影响效率的瓶颈
- 给出一套更低人力负担的改进建议

==================================================
一、当前已知流程背景
==================================================

当前项目已经形成以下模式：

1. 主控官（Codex）不再默认亲自做主实现
2. 主控官负责：
   - 冻结模块边界
   - 拆分子任务
   - 分派给 AI 军团
   - 回收 execution / review / compliance
   - 做统一终验
3. AI 军团负责：
   - Execution
   - Review
   - Compliance
4. 所有模块都按三权分立推进：
   - Execution
   - Review
   - Compliance
5. 所有任务都要求：
   - 标准写回路径
   - execution_report
   - review_report
   - compliance_attestation
6. 主控官终验前必须确认：
   - 回收件齐全
   - 并行/串行依赖满足
   - 无阻断性越界

==================================================
二、当前已经暴露的问题
==================================================

当前最大的痛点不是“不会拆任务”，而是：

1. 主控官体力劳动太大
2. 常常要人工确认：
   - 谁该接单
   - 谁该审查
   - 谁该合规
   - 哪个文件必须写回
   - 哪个任务可以并行，哪个必须串行
   - 哪些回收件缺失
3. 实际上很多 review / compliance 已经做了，但如果没有写回标准文档，就会形成缺件
4. 一旦任务书没有把：
   - 写回路径
   - 审查提示词
   - 合规提示词
   - 并行/串行依赖
   写死，就会导致主控官多一轮追收和解释

==================================================
三、当前已有的改进方向
==================================================

当前已经开始做两件事：

1. 把“主控官如何拆分 / 分派 / 回收 / 终验”的方法，固化成一个 skill
2. 计划把“AI 军团任务投递 / 写回 / 状态流转 / 升级到主控官”的协议单独模块化

当前的思考是：

- 短期先用 skill 固化方法
- 中期用协议文档固化流程
- 后期再用代码自动化：
  - task dispatch
  - writeback
  - state machine
  - backfill detection
  - final gate aggregation

==================================================
四、你要回答的问题
==================================================

请你只回答以下问题，不要跑题：

### 1. 这套“主控官 + AI 军团 + 三权分立 + 标准写回 + 统一终验”的模式，
从专业角度看是否成立？

请回答：
- 成立 / 基本成立 / 不成立
- 为什么

### 2. 这套流程最大的结构性瓶颈是什么？

请明确指出最关键的 3 个瓶颈。

### 3. 当前最该先补的，是：
- skill
- 协议文档
- 代码自动化工具
- 还是别的

请给出优先级排序。

### 4. “AI 军团任务投递与自动流转协议”最小应该包含哪些东西？

请给出一个最小协议清单，至少覆盖：
- 任务投递
- 状态流转
- 标准写回
- 自动接力
- 升级到主控官

### 5. 如果目标是让人类只做三件事：
- 给模块文档
- 下发给 AI 军团
- 确认最终结果

那么这套流程还缺哪一层？

### 6. 你建议的下一步最合理模块是什么？

请给出一个模块名称，并说明其唯一目标。

==================================================
五、输出格式要求
==================================================

请严格按以下结构输出：

1. 总体判断
2. 三个核心瓶颈
3. 优先级排序
4. 最小协议清单
5. 缺失的关键层
6. 下一步建议模块
7. 风险提醒

要求：
- 直说，不要客套
- 以架构师 / 流程设计师视角回答
- 不要泛泛而谈
- 不要建议“先观察再说”
- 给出明确结论
```

## 推荐用法

如果你要拿去做真正的 AI 会议，建议：

1. 先把这份提示词发给 Claude
2. 再把 Claude 的回答贴回给 Codex
3. 让 Codex做：
   - 差异比对
   - 吸收可用建议
   - 生成下一模块主控提示词
