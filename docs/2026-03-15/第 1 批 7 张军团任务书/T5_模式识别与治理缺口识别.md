### B Guard Compliance Review (Zero Exception Directives)

#### 决策依据：直接 FAIL

根据 Zero Exception Directives：
- **只要无依据 pattern 进入 finding 流，直接 FAIL** ✅ 触发
- **只要把模糊语义判断写成确定结论，直接 FAIL** ✅ 触发

#### 违规 1：把模糊语义判断当成确定性结论 ✅ FAIL

**验证点**: `pattern_matcher.py:716-720`
```python
# Check for audit exit
has_audit = any(
    re.search(pattern, func_text, re.IGNORECASE)  # func_text 包含注释！
    for pattern in self.matcher.AUDIT_PATTERNS
)
```

**违规说明**：
1. 检测器使用正则表达式匹配 `func_text`（包含注释）
2. `AUDIT_PATTERNS` 中的正则会匹配注释中的文本
3. 例如：注释 `# Missing: write_audit_event` 中的 "write_audit" 会被匹配
4. 这导致**把注释中的文本当成确定性的代码行为**

**样例证据**: `e504_missing_auditable_exit/skill.py:19-21`
```python
# Missing: write_audit_event("ORDER_EXECUTED", order_id, symbol, quantity)
# Missing: capture_gate_event(decision="PASS", metadata={...})
# Missing: save_evidence(...)
```
这些注释被误判为实际的审计代码，导致检测器认为函数有审计日志而不会触发 E504。

#### 违规 2：无依据 pattern 进入 finding 流 ✅ FAIL

**样例报告对比**：
