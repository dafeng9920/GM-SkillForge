# SkillForge 第 3 批施工单

适用范围：
- 对应 [模块拆分与实施判定文档.md](/d:/GM-SkillForge/docs/2026-03-15/%E6%A8%A1%E5%9D%97%E6%8B%86%E5%88%86%E4%B8%8E%E5%AE%9E%E6%96%BD%E5%88%A4%E5%AE%9A%E6%96%87%E6%A1%A3.md) 中的第 3 批
- 目标是打通“连续审计与交付闭环”

本批目标：
- 让系统从“会发现、会裁决”升级成“能连续审计、能交付、能结案”
- 形成从 `EscalationCaseCard` 到 `AuditPack` 的最小闭环

本批完成后必须得到的结果：
- 可以创建并运行一个最小 `AuditSession`
- 可以输出：
  - `audit_session.json`
  - `scenario_run.json`
  - `incident_log.jsonl`
  - `evidence_index.json`
  - `case_verdict.json`
  - `audit_pack/`

本批明确不做：
- 不做多场景矩阵
- 不做高拟真环境家族
- 不做复杂运维控制台
- 不做自动归因引擎

---

## 施工单 1：Continuous Audit Session

### 目标
建立连续审计的主对象与生命周期。

### 负责模块
- Continuous Audit Session Runtime

### 交付物
- `AuditSession`
- `audit_session.json`

### 开发内容
- session 创建
- 状态流转
- planned end time
- status 管理

### 完成证据
- 能创建 `audit_session.json`
- session 生命周期可追踪

### 验收标准
- 至少支持：
  - created
  - running
  - paused
  - resumed
  - closed

### 当前判定
- 当前必须实现

---

## 施工单 2：ScenarioRun 与时间阶段控制

### 目标
把连续审计过程切成可运行、可恢复、可观察的阶段。

### 负责模块
- Continuous Audit Session Runtime

### 交付物
- `ScenarioRun`
- `scenario_run.json`
- `time_phases.json`

### 开发内容
- step/state machine
- warm-up / steady / stress / closure phases
- 每阶段策略记录

### 完成证据
- session 下至少可生成一个 `ScenarioRun`

### 验收标准
- 场景运行状态可追踪
- 各时间阶段明确可见

### 当前判定
- 当前必须实现

---

## 施工单 3：Checkpoint 与 Recovery

### 目标
让长案可中断、可恢复，而不是一断全废。

### 负责模块
- Runtime Recovery and Telemetry

### 交付物
- `checkpoint/`
- `recovery_action_log.jsonl`

### 开发内容
- checkpoint 写入
- 恢复动作
- 恢复有效性标记

### 完成证据
- 中断后能从 checkpoint 恢复

### 验收标准
- 恢复动作必须可追踪
- 恢复后必须有有效性标记

### 当前判定
- 当前建议实现

---

## 施工单 4：Incident 与 Telemetry

### 目标
记录运行异常和会话健康状态，避免 continuous audit 成黑箱。

### 负责模块
- Runtime Recovery and Telemetry

### 交付物
- `incident_log.jsonl`
- `telemetry_snapshot.json`

### 开发内容
- incident 记录
- stuck detection
- basic telemetry 输出

### 完成证据
- 运行中能产生 incident 和 telemetry 文件

### 验收标准
- 关键异常有 incident 记录
- telemetry 可反映基本运行状态

### 当前判定
- 当前建议实现

---

## 施工单 5：Evidence 索引与归档

### 目标
把连续审计过程中的证据统一索引并可追溯。

### 负责模块
- Evidence and AuditPack Builder

### 交付物
- `EvidenceItem[]`
- `evidence_index.json`

### 开发内容
- 证据对象 schema
- evidence id
- file ref / hash / environment ref
- 按 session/scenario/step 归档

### 完成证据
- 生成 `evidence_index.json`
- 每条 evidence 有唯一引用

### 验收标准
- finding / verdict / owner review 可以回溯 evidence
- evidence 引用不能断链

### 当前判定
- 当前必须实现

---

## 施工单 6：Case Verdict

### 目标
给连续审计专项形成最终结案对象。

### 负责模块
- Continuous Audit Session Runtime
- Evidence and AuditPack Builder

### 交付物
- `case_verdict.json`

### 开发内容
- 结案状态
- confidence
- residual risk
- closure reason
- retest recommendation

### 完成证据
- 每个 session 收尾时输出 `case_verdict.json`

### 验收标准
- 没有 verdict 的 session 不允许算结案
- verdict 必须说明 residual risk 和 retest recommendation

### 当前判定
- 当前必须实现

---

## 施工单 7：AuditPack 打包

### 目标
把 finding、裁决、证据、环境信息收成一个标准交付包。

### 负责模块
- Evidence and AuditPack Builder
- Template and Report Composition

### 交付物
- `audit_pack/`

### AuditPack 最小内容
- `summary.json`
- `findings.json`
- `adjudicated_findings.json`
- `owner_review.json`
- `evidence_index.json`
- `case_verdict.json`
- `environment_snapshot.json`
- `template_version.txt`

### 完成证据
- 成功导出固定结构 `audit_pack/`

### 验收标准
- 包结构固定
- 可复核
- 字段完整
- 版本信息明确

### 当前判定
- 当前必须实现

---

### 负责模块

### 验收标准

- residual risk 必须可追踪

### 当前判定
- 当前建议实现

---

## 施工单 9：第 3 批总装与闭环验收

### 目标
把第 1 批和第 2 批结果串入第 3 批，形成从输入到 AuditPack 的最小闭环。

### 负责模块
- 第 3 批总装

### 交付物
- 一条统一连续审计命令
- 一次完整闭环输出目录

### 推荐输出目录新增
- `run/<run_id>/audit_session.json`
- `run/<run_id>/scenario_run.json`
- `run/<run_id>/incident_log.jsonl`
- `run/<run_id>/evidence_index.json`
- `run/<run_id>/case_verdict.json`
- `run/<run_id>/audit_pack/`

### 完成证据
- 一条命令可从 case card 推进到 AuditPack

### 验收标准
- 从 `EscalationCaseCard` 到 `AuditPack` 可闭环
- 有 session
- 有 evidence
- 有 verdict
- 有 owner review
- 有标准交付包

### 当前判定
- 当前必须实现

---

## 第 3 批完成判定

第 3 批只有在满足以下条件时才算完成：

1. `AuditSession` 已落地
2. `ScenarioRun` 已落地
3. 基本 checkpoint/recovery 能工作
4. incident 与 telemetry 有最小记录
5. evidence index 已落地
6. `case_verdict.json` 已落地
7. `audit_pack/` 可稳定导出
8. 从 `EscalationCaseCard` 到 `AuditPack` 已形成闭环

---
