# SkillForge 第 1 批施工单

适用范围：
- 对应 [模块拆分与实施判定文档.md](/d:/GM-SkillForge/docs/2026-03-15/%E6%A8%A1%E5%9D%97%E6%8B%86%E5%88%86%E4%B8%8E%E5%AE%9E%E6%96%BD%E5%88%A4%E5%AE%9A%E6%96%87%E6%A1%A3.md) 中的第 1 批
- 目标是先打通“发现链”，不进入裁决链和连续审计链

本批目标：
- 让系统具备稳定发现一小类高价值缺陷的能力
- 形成从输入到 `findings.json` 的最小闭环
- 为第 2 批裁决链提供稳定输入

本批完成后必须得到的结果：
- 可以对一个固定 skill/repo 目标执行一次确定性发现流程
- 可以输出标准化的：
  - `intake_request.json`
  - `normalized_skill_spec.json`
  - `validation_report.json`
  - `rule_scan_report.json`
  - `pattern_detection_report.json`
  - `findings.json`

本批明确不做：
- 不做 Finding Adjudication
- 不做 Owner Review
- 不做 Escalation Gate
- 不做 Continuous Audit Runtime
- 不做 AuditPack

---

## 施工单 1：固定输入入口

### 目标
实现唯一、确定性的审计输入入口，锁死 `intent_id + repo_url + commit_sha`。

### 负责模块
- Intent Intake Gateway

### 交付物
- CLI 或 API 入口
- `IntakeRequest` 数据结构
- `intake_request.json`

### 开发内容
- 定义输入对象 schema
- 强制 `repo_url + commit_sha`
- 拒绝 branch-only 输入
- 拒绝未知 `intent_id`

### 完成证据
- 能运行：
  - `skillforge run audit_repo_skill --repo <url> --commit <sha>`
- 生成固定格式 `intake_request.json`
- 缺失 `commit_sha` 时命令失败

### 验收标准
- 相同输入多次执行，`intake_request.json` 完全一致
- 非法输入返回固定错误

### 当前判定
- 当前必须实现

---

## 施工单 2：Skill 解析与标准化

### 目标
把目标 repo 中的 skill/manifest/依赖结构解析成统一对象。

### 负责模块
- Skill Parsing and Normalization

### 交付物
- `NormalizedSkillSpec`
- `DependencyGraph`
- `ManifestSnapshot`
- `normalized_skill_spec.json`

### 开发内容
- 解析当前仓库约定的 skill 目录结构
- 读取 manifest
- 提取依赖、输入输出合同、边界描述
- 标准化字段命名

### 完成证据
- 能对现有样例 skill 输出 `normalized_skill_spec.json`
- 缺 manifest 的样例稳定失败

### 验收标准
- 合法样例可解析
- 非法样例可定位失败原因
- 输出字段完整率 100%

### 当前判定
- 当前必须实现

---

## 施工单 3：合同与结构校验

### 目标
先拦掉明确的结构缺陷和合同缺陷。

### 负责模块
- Contract and Schema Validation

### 交付物
- `validation_report.json`
- `validation_failures[]`

### 开发内容
- schema 校验
- contract 校验
- 输入输出字段一致性检查
- 必填项检查
- 错误码和字段路径输出

### 完成证据
- 提供至少 3 组样例：
  - 正常 skill
  - 缺字段 skill
  - contract 冲突 skill
- 每组样例都有固定结果

### 验收标准
- 合法 skill 不误报
- 非法 skill 必拦截
- 每个失败项都有：
  - 错误码
  - 字段路径
  - 修复建议

### 当前判定
- 当前必须实现

## 施工单 4：规则与边界扫描

### 目标
通过确定性规则发现高价值规则缺陷和边界缺口。

### 负责模块
- Rule and Boundary Scanning

### 交付物
- `rule_scan_report.json`
- 初始 `raw_findings[]`

### 开发内容
- 敏感权限扫描
- 外部动作扫描
- 危险模式扫描
- 边界缺失扫描
- 规则版本记录

### 完成证据
- 提供固定规则样例库
- 对每个样例给出预期命中结果

### 验收标准
- 规则样例命中结果稳定
- 每条命中都可定位源位置
- 每条命中都能追溯规则版本

### 当前判定
- 当前必须实现

## 施工单 5：模式识别与治理缺口识别

### 目标
把多个确定性信号组合成更高价值的风险模式。

### 负责模块
- Risk Pattern Detection

### 交付物
- `pattern_detection_report.json`
- `pattern_findings[]`

### 开发内容
- Anti-pattern library
- Governance gap detector
- 少量高价值模式匹配

### MVP 限定
- 只做少量模式
- 优先覆盖：
  - 高权限调用未限界
  - 外部动作无 stop rule
  - retry 缺幂等保护
  - 缺少可审计出口

### 完成证据
- 模式样例库
- 每个模式样例都有固定输出

### 验收标准
- 至少覆盖当前锁定缺陷中的一组高价值模式
- 每个模式 finding 必须能解释来源依据

### 当前判定
- 当前必须实现

## 施工单 6：Finding 结构化

### 目标
把 validation、rule scan、pattern detection 的结果统一成稳定 finding 对象。

### 负责模块
- Finding Construction

### 交付物
- `findings.json`

### 开发内容
- 定义统一 finding schema
- 绑定 evidence refs
- 初始 severity
- 初始 confidence
- finding id 生成规则

### finding 最小字段
- `finding_id`
- `title`
- `category`
- `source_layer`
- `severity`
- `confidence`
- `evidence_refs`
- `rule_or_pattern_ref`
- `suggested_fix`

### 完成证据
- 输出固定格式 `findings.json`
- 每条 finding 均可追溯到输入来源

### 验收标准
- 无裸 finding
- 每条 finding 都能追溯：
  - validation failure 或 rule hit 或 pattern hit
- 字段完整率 100%

### 当前判定
- 当前必须实现

## 施工单 7：第 1 批总装与回归样例

### 目标
把前 6 张施工单串成一次可运行、可回归的发现链。

### 负责模块
- 第 1 批总装

### 交付物
- 一条统一命令
- 一组固定测试样例
- 一次完整流水线输出目录

### 开发内容
- 把输入 -> 解析 -> 校验 -> 扫描 -> 模式 -> finding 串起来
- 固定输出目录结构
- 准备最少 3 组回归样例

### 推荐输出目录
- `run/<run_id>/intake_request.json`
- `run/<run_id>/normalized_skill_spec.json`
- `run/<run_id>/validation_report.json`
- `run/<run_id>/rule_scan_report.json`
- `run/<run_id>/pattern_detection_report.json`
- `run/<run_id>/findings.json`

### 完成证据
- 一条命令可以跑通全链
- 对三组样例都能稳定输出

### 验收标准
- 主链无人工拼接
- 回归样例重复执行结果稳定
- 任何一步失败都能定位失败位置和错误码

### 当前判定
- 当前必须实现

## 第 1 批完成判定

第 1 批只有在满足以下条件时才算完成：

1. 已存在唯一统一入口，且强制 `repo_url + commit_sha`
2. 已能输出稳定的 `normalized_skill_spec.json`
3. 已能输出 `validation_report.json`
4. 已能输出 `rule_scan_report.json`
5. 已能输出 `pattern_detection_report.json`
6. 已能输出结构稳定、证据可追溯的 `findings.json`
7. 已有至少 3 组固定回归样例
8. 一条命令可跑通全链

只要以上 8 条有任意一条不成立，第 1 批就不能算完成。
