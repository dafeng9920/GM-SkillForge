# Quant 架构决策：回归核心

> 日期: 2026-02-23
> 决策: 路线 B - Quant 作为 Adapter 层
> 状态: **已确定**

---

## 1. 决策

**Quant 只做 adapter（策略适配），执行交给 SkillForge 核心 Skills，放弃独立部署。**

```
┌─────────────────────────────────────────────────────────────┐
│                    SkillForge 核心                          │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ Gate 引擎   │  │ 审计服务    │  │ 证据链管理  │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 执行 Skills │  │ 风控 Skills │  │ 治理 Skills │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
│                                                             │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           │ 标准接口
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                 Quant Adapter Layer                         │
│                                                             │
│  ┌─────────────────┐  ┌─────────────────┐                  │
│  │ 策略适配器       │  │ 数据源适配器     │                  │
│  │ - stock-selector │  │ - OpenBB        │                  │
│  │ - ETF轮动        │  │ - Kronos        │                  │
│  │ - 其他策略       │  │ - 其他数据源     │                  │

│                                                             │
│  职责：翻译，不执行                                          │
│  - 把量化概念 → SkillForge 标准格式                          │
│  - 把策略信号 → 核心 Skills 可理解的输入                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 2. 职责划分

### 2.1 Quant Adapter Layer（只做翻译）

| 适配器类型 | 职责 | 不做 |
|------------|------|------|
| 策略适配器 | 策略逻辑 → 标准信号格式 | 不执行交易 |
| 数据源适配器 | 数据获取 → 标准 Evidence | 不存储数据 |
| 模型适配器 | 预测结果 → 标准输出 | 不做风控判断 |

### 2.2 SkillForge 核心（所有执行）

| 核心层 | 职责 |
|--------|------|
| Gate 引擎 | 所有 Gate 检查 |
| 执行 Skills | 所有交易执行 |
| 风控 Skills | 所有风险控制 |
| 审计服务 | 所有日志/证据链 |

## 5. 新架构下的数据流

```
策略代码                    Adapter                    SkillForge 核心
   │                          │                            │
   │ 1. 策略计算              │                            │
   ├─────────────────────────►│                            │
   │                          │ 2. 翻译为标准格式           │
   │                          ├───────────────────────────►│
   │                          │                            │ 3. Gate 检查
   │                          │                            │ 4. 风控判断
   │                          │                            │ 5. 执行交易
   │                          │                            │ 6. 记录审计
   │                          │◄───────────────────────────┤
   │                          │ 7. 返回执行结果             │
   │◄─────────────────────────┤                            │
   │ 8. 策略收到结果          │                            │
```

---

## 6. 实施原则

1. **先定好，后不乱** - 架构边界清晰后再写代码
2. **Adapter 只翻译** - 不做任何执行决策
3. **核心管一切** - 所有执行、风控、审计都在核心层
4. **废弃独立部署** - 量化相关代码合并到主仓库
