- 商用版 skill：允许用户填入自身环境参数，让 skill 吸收并转化为贴合本地系统架构的运行版本
- 次日验收 / 夜间执行

### L5 的云端工作流 skill 原则
- 允许逐步迭代，不追求一步到位
- 每一版都必须可复用，不能是一次性工程
- 目标是做成长期复用、持续演进的底层工作流 skill
- 用户侧只填自己的环境参数、仓库结构、执行约束、部署边界
- skill 负责把这些输入吸收、转化，并适配到用户自己的本地/云端系统架构
- 正式出品的商用版 skill，必须先经过系统自身审计，再允许出品/入库/放行
- 也就是说，出品不是“生成即上线”，而是“生成 -> 自审计 -> 通过后出品”
- 因此最终交付给用户的 skill，自带内部质量保证，不是裸产物
- 这类底层 workflow skill 不是静态成品，而是会随着审计规则迭代、用户环境变化、模型版本更新而持续升级
- V1 先解决“云端替人跑一夜”
- V2 再解决“自动执行 + 自动审查 + 自动合规 + 自动测试 + 自动回写”
- V3 再解决“自动沉淀成 prototype / skill candidate”
- 终态不是临时脚本，而是可商用、可配置、可迁移的治理工作流 skill
