# constitution_v1

## 1. 文档定位
`constitution_v1.md` 是系统最高约束文本。  
它只定义边界、裁决与通过条件，不定义实现细节。

## 2. 最高信条
1. 终端事实优先：系统只以可验证输出为事实来源。  
2. Fail-Closed：关键条件缺失或校验失败时，必须拒绝通过。  
3. 失败不可跳过：任何失败都必须被记录、可追溯、可复核。  
4. 证据先于解释：解释不能覆盖证据。  
5. 宪法先于能力：先有约束，后有功能。

## 3. 不可回退红线
1. 不允许以叙事替代证据。  
2. 不允许把失败标记为“部分成功”。  
3. 不允许在无证据链的情况下执行关键动作。  
4. 不允许通过人工口径绕过 Gate。  
5. 不允许旧代码复制进入新实现。

## 4. 裁决优先级
优先级从高到低如下：  
`Constitution > Contracts > Acceptance Gates > Runtime Decisions > Notes`

若下层规则与上层冲突，以下层无效处理。

## 5. 系统责任边界
1. 宪法负责“可否”。  
2. 契约负责“如何判定”。  
3. 验收负责“是否达标”。  
4. 运行时负责“按规则执行”，不拥有改写规则的权限。

## 6. 最小治理闭环（意图定义）
`终端事实 -> Evidence -> Replay -> Gate Verdict`

任一环断裂，整体判定为 `REJECTED`。

## 7. 通过与拒绝定义
1. `PASSED`：全部硬门禁通过，且追溯链完整。  
2. `REJECTED`：任一硬门禁失败、缺失、不可验证、不可回放。  
3. 不存在 `WARNING` 作为放行依据。

## 8. 变更原则
1. 宪法变更只能增强约束，不能放宽红线。  
2. 任何契约或验收文档变更，必须引用本宪法条款。  
3. 未引用宪法依据的规则变更视为无效。

## 9. 生效声明
本文档生效后，所有架构、契约、验收与实现必须服从本文。  
与本文冲突的既有描述自动降级为参考材料。
