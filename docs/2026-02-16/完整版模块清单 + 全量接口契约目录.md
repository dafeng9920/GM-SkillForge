# 外螺旋完整版模块清单 + 全量接口契约目录

> 文档日期: 2026-02-16
> 适用范围: 自然语言 -> 外螺旋智能编排 -> Skill 产出 全链路（完整版）
> 核心原则: 终端事实优先、Fail-Closed、治理内核不可 Skill 化

---

## 0. 架构总线与边界

系统目标链路:

```text
Natural Language Input
  -> Intent Ingest
  -> Negative Convergence
  -> Intent Contract Freeze
  -> Intelligent Composer
  -> Skill Draft
  -> Skill Artifact Build
  -> Gate Checks
  -> Evidence Write
  -> Replay Verify
  -> Pass / Reject
```

边界结论:
- 可 Skill 化: 外螺旋认知与编排能力
- 不可 Skill 化: Gate / Evidence / Replay / Policy Loader / Immutable Audit
- 任何 Skill 不能绕过治理内核

---

## 1. 完整版模块清单

### 1.1 L0 接入层（Input & Session）

#### `input-gateway`
- 职责: 接收自然语言输入、会话元信息、用户上下文
- 输入: `UserInputEnvelope`
- 输出: `IntentIngestRequest`
- 约束: 不做决策、不做编排

#### `session-context-store`
- 职责: 维护会话状态、轮次、用户否定反馈历史
- 输入: `SessionPatch`
- 输出: `SessionSnapshot`
- 约束: 只存事实，不存模型推断结论

---

### 1.2 L1 外螺旋认知层（Skill 化）

#### `skill.intent_ingest`
- 职责: NL 输入结构化解析（初始意图、约束、非目标）
- 输入: `IntentIngestRequest`
- 输出: `IntentParseResult`
- 失败策略: 字段不全则 `REJECT_WITH_CLARIFICATION`

#### `skill.negative_convergence`
- 职责: 否定式收敛（用户否定反馈驱动意图逼近）
- 输入: `ConvergenceRequest`
- 输出: `ConvergenceResult`
- 失败策略: 收敛置信不足则 `NEED_MORE_NEGATIVE_FEEDBACK`

#### `skill.intent_contract_freezer`
- 职责: 生成并冻结 `IntentContract`
- 输入: `ContractFreezeRequest`
- 输出: `IntentContract`
- 失败策略: 缺少成功标准/约束时拒绝冻结

---

### 1.3 L2 智能编排层（Skill 化）

#### `skill.composer`
- 职责: 基于 `IntentContract` 生成 `SkillDraft`
- 输入: `ComposerRequest`
- 输出: `ComposerResult`
- 约束: 禁止修改 `IntentContract` 冻结字段

#### `skill.risk_tradeoff_planner`
- 职责: 生成可比较方案（fast/steady/power 等策略）
- 输入: `TradeoffPlanRequest`
- 输出: `TradeoffPlanResult`
- 约束: 仅建议，不授权执行

#### `skill.skill_artifact_builder`
- 职责: 将 `SkillDraft` 编译为 `SkillArtifact`
- 输入: `SkillArtifactBuildRequest`
- 输出: `SkillArtifact`
- 约束: 必须携带 `contract_ref` + `decision_hash`

---

### 1.4 L3 治理内核（不可 Skill 化）

#### `gate-engine`
- 职责: Preflight/Evidence/Replay 门禁
- 输入: `GateCheckRequest`
- 输出: `GateVerdict`
- 约束: 缺字段一律拒绝（Fail-Closed）

#### `evidence-writer`
- 职责: 终端事实与过程事实写入 Evidence
- 输入: `EvidenceWriteRequest`
- 输出: `EvidenceWriteResult`
- 约束: 先写证据再决策

#### `replay-verifier`
- 职责: 基于证据与快照做可重放验证
- 输入: `ReplayRequest`
- 输出: `ReplayResult`
- 约束: 不能调用推导器补事实

#### `policy-loader-immutable`
- 职责: 加载不可变治理规则
- 输入: `PolicyLoadRequest`
- 输出: `PolicySnapshot`
- 约束: 运行期不可被 Skill 覆盖

#### `audit-chain-store`
- 职责: 审计链只追加写入
- 输入: `AuditEvent`
- 输出: `AuditAck`
- 约束: 不允许更新历史事件

---

### 1.5 L4 运行与发布层

#### `skill-runtime-host`
- 职责: 调度 Skill 执行，记录 trace
- 输入: `SkillRunRequest`
- 输出: `SkillRunResult`
- 约束: 所有执行调用必须经过 gate-engine

#### `skill-registry`
- 职责: SkillArtifact 版本、签名、回滚索引
- 输入: `ArtifactRegisterRequest`
- 输出: `ArtifactRegisterResult`
- 约束: 不存未通过 gate 的 artifact

## 3. 核心对象契约（必须先冻结）

### 3.1 `IntentContract`（冻结后不可变）

字段:
- `contract_id: string`
- `session_id: string`
- `turn_id: number`
- `goal: string`
- `constraints: string[]`
- `non_goals: string[]`
- `success_criteria: string[]`
- `rollback_policy: object`
- `risk_level: enum(low|medium|high|critical)`
- `frozen_at: datetime`
- `contract_hash: string`

规则:
- 缺 `success_criteria` 不可冻结
- 冻结后只能新版本，不可原地修改

### 3.2 `SkillDraft`

字段:
- `draft_id: string`
- `contract_ref: string`
- `steps: Step[]`
- `required_inputs: FieldSpec[]`
- `expected_outputs: FieldSpec[]`
- `risk_notes: string[]`
- `decision_hash: string`

规则:
- `contract_ref` 必填
- `steps` 不得为空

### 3.3 `SkillArtifact`

字段:
- `artifact_id: string`
- `draft_ref: string`
- `version: string`
- `entrypoint: string`
- `schema: object`
- `signature: string`
- `build_meta: object`

规则:
- 未通过 Gate 不可注册
- 必须能回溯到 `contract_ref`

### 3.4 `EvidencePackRef`

字段:
- `evidence_pack_ref: string`
- `trace_id: string`
- `session_id: string`
- `turn_id: number`
- `spec_ref: object`
- `decision_hash: string`
- `execution_summary: object`
- `integrity: enum(OK|FAILED)`
- `created_at: datetime`

规则:
- `integrity != OK` 必须阻断后续 run/publish

### 3.5 `ReplayManifestRef`

字段:
- `replay_id: string`
- `session_id: string`
- `turn_seq: number`
- `spec_ref: object`
- `decision_hash: string`
- `replay_verified: boolean`
- `replayed_at: datetime`

规则:
- `replay_verified=false` 必须 Gate 拒绝

## 4. 关键流程接口图（完整版）

### 4.1 外螺旋收敛流程

1. `POST /v1/input/ingest` -> `IntentParseResult`
2. `POST /v1/outer/converge` -> `ConvergenceResult`
3. `POST /v1/outer/contracts/freeze` -> `IntentContract`

### 4.2 编排与产物流程

1. `POST /v1/composer/compose` -> `SkillDraft`
2. `POST /v1/composer/tradeoff-plan` -> `TradeoffPlanResult`
3. `POST /v1/skills/build` -> `SkillArtifact`
4. `POST /v1/skills/register` -> `ArtifactRegisterResult`

### 4.3 治理与审计流程

1. `POST /v1/gates/preflight/check`
2. `POST /v1/evidence/write`
3. `POST /v1/gates/evidence/check`
4. `POST /v1/replay/verify`
5. `POST /v1/gates/replay/check`
6. `POST /v1/audit/events/append`

---

## 5. 状态机（完整版）

### 5.1 Contract 状态机

`DRAFT -> CANDIDATE -> FROZEN -> SUPERSEDED`

规则:
- 仅 `FROZEN` 可进入 composer
- 变更需求必须生成新 contract 版本

### 5.2 SkillArtifact 状态机

`BUILT -> GATE_PASSED -> REGISTERED -> ACTIVE -> DEPRECATED`

规则:
- `BUILT` 不能直接 `ACTIVE`
- 缺 Evidence/Replay 不能 `REGISTERED`

### 5.3 Run 状态机

`REQUESTED -> PREFLIGHT_OK -> EVIDENCE_OK -> REPLAY_OK -> EXECUTED`

失败路径:
- 任意门禁失败 -> `REJECTED`

---

## 6. 非功能与治理约束（必须实现）

- 全接口必须带 `trace_id`、`session_id`、`turn_id`
- 全写操作必须落审计事件
- Hash 采用稳定 canonical JSON 算法
- 存储策略: 审计与证据采用 append-only
- Fail-Closed: 缺字段、版本漂移、完整性失败一律拒绝

---

## 7. 明确禁止项（避免范围漂移）

当前完整版架构中，以下能力不在“外螺旋 -> Skill 产出”主线内：
- n8n workflow 运行编排
- 自动重试自动修复
- 模型自解释覆盖终端事实
- 任何绕开 Gate/Evidence/Replay 的快捷通道

---

## 8. 实施顺序（完整版落地建议）

1. 先冻结三大契约: `IntentContract` / `SkillDraft` / `SkillArtifact`
2. 上线治理内核四件套: `gate-engine` / `evidence-writer` / `replay-verifier` / `audit-chain-store`
3. 接入外螺旋 Skill 链: ingest -> converge -> freeze -> compose -> build
4. 打通注册与运行门禁
5. 完成 replay 回归用例库

## 10. 一句话结论

外螺旋可以完整 Skill 化，但治理内核必须保持宿主级不可绕过能力；只有这样，`自然语言 -> 智能编排 -> Skill 产出` 才是可审计、可回放、可拒绝的完整版链路。
