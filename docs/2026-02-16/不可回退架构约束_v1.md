# 不可回退架构约束 v1

> Constitution SSOT: `docs/2026-02-16/constitution_v1.md`

## 0. 文档目的
固化 GM-SkillForge 当前阶段的三条最高架构约束，作为实施与审计的不可回退边界。

## 1. 宪法最高层（不可绕过）
1. 宪法是内核最终裁决依据，必须版本化并写入 `provenance` / `AuditPack`（`constitution_ref` + `constitution_hash`）。
2. 宪法覆盖点至少包括：
- `constitution_risk_gate`
- 发布/下架（`tombstone`）约束
3. n8n 与 Skills 只能“提议/产证据”，不能“最终裁决”。

## 2. 核心 Skill（v0 最小集）
以下 Skill 为 v0 最小可用集合，缺失任一项不得宣称内核可用：
1. `scan_skill_shape`：识别“什么是 skill”，产出 `fit_score`。
2. `static_denylist_scan`：静态危险模式扫描，规则必须版本化。
3. `spec_drafter_<intent>`：意图到合同草案（`contracts-first`）。
4. `scaffold_python`：Python 模板生成。
5. `pack_builder_helper`：协助清点 pack 文件与证据引用，但不做达标裁决。

## 3. n8n 分层策略（n8n 仅编排）
1. n8n = Orchestrator：
- 触发
- 路由
- 重试
- 人工审批
- 通知
- 外部系统集成
2. SkillForge 内核 = Governor：
- 8 Gate 执行
- `EvidenceRef`
- `L3 AuditPack`
- RAG（`at-time` / `tombstone`）
- 最终 `PASS/FAIL` 裁决
3. 强制输入：
- `repo_url`
- `commit_sha`
- 可选 `at_time`
4. 禁止项：
- 禁止 n8n 以 `latest` 自行决定输入版本。
- 禁止 n8n 内建治理裁决逻辑。
5. 交互约束：
- n8n 每一步必须调用 SkillForge 内核接口（如 `run_intent` / `fetch_pack` / `query_rag`）。

## 4. 违规判定（Fail-Closed）
任一条违反即 `REJECTED`，不得“部分通过”：
1. 缺失宪法版本引用或哈希。
2. 由 n8n 或普通 Skill 做最终裁决。
3. 使用 `latest` 替代 `commit_sha`。
4. 核心 Skill v0 集合不完整却继续发布。

## 5. 生效与优先级
1. 本文档生效后，任何设计/实现/编排与本文冲突时，以本文为准。
2. 优先级：`Constitution > 不可回退架构约束 v1 > 其他设计文档/执行计划`。

## 6. V1-Prove 窗口与冻结机制
1. 窗口期：`2026-02-17` 至 `2026-03-03`（14 天）。
2. 窗口期内仅允许“证据驱动修订”：
- 只改被真实失败案例击中的条款。
- 每次修订必须绑定 `evidence_ref`。
3. 开发期提醒（防绕过）：
- 未通过 `constitution_risk_gate` 不得进入实现。
- 缺 `constitution_ref/hash` 不得进入实现。
- 使用 `latest` 代替 `commit_sha` 直接 `REJECTED`。
4. 冻结到位条件（全部满足）：
- 至少 1 个真实 `REJECTED` 案例可追溯。
- 至少 1 个真实 `PASSED` 案例可追溯。
- 修订记录全部可关联到对应 `evidence_ref`。
5. 到期动作：
- 满足到位条件则冻结为 `V1.0 Final`。
- 冻结后非安全问题不得修改宪法层条款。
